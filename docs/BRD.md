# Business Requirements Document (BRD)

## Project Name
IT Service Request & Incident Management

## Business Problem

Organizations require a centralized system to manage IT service requests and incident tickets efficiently. Manual ticket handling can lead to delays, inconsistent prioritization, and lack of visibility.

## Business Objective

Develop a ServiceNow application that enables users to:

- Create IT Service Requests
- Create Incident Tickets
- Automatically calculate ticket priority
- Automatically assign tickets to support groups
- Track ticket status
- Notify users through email automation

## Scope

### In Scope

- Ticket creation
- Incident management
- Service request management
- Priority calculation
- Assignment group management
- Email notifications
- Resolution tracking

### Out of Scope

- SLA Management
- CMDB Integration
- Knowledge Base Integration
- Service Portal

## Stakeholders

| Role | Responsibility |
|--------|---------------|
| End User | Raise tickets |
| IT Support Team | Resolve tickets |
| Service Desk Manager | Monitor operations |
| System Administrator | Maintain application |

## Expected Benefits

- Faster ticket handling
- Improved visibility
- Automated workflows
- Reduced manual effort
- Better user experience

## Success Criteria

- Ticket creation successful
- Priority calculated automatically
- Assignment group populated automatically
- Email notifications triggered
- Resolution notes captured before closure
